# P3R UI 动效拆解与复刻规范

这份文档聚焦《Persona 3 Reload》菜单 UI 的动效语言，重点展开三部分：

1. 角色入场动作时间线
2. 界面分层顺序
3. 网页 / CSS / AE 的复刻方法

目的不是做一份“考据型百科”，而是把 P3R 的感觉拆成可执行的设计规则。

## 0. 依据与边界

### 0.1 可确认的信息

以下信息可以从公开采访或技术拆解中确认：

- P3R 的菜单核心意象是水、泡、玻璃反射和“沉入内心”
- 主角在菜单中被设计成“漂浮 / 下沉”的状态
- 菜单里的主角使用了 UI 专用 3D 模型
- 不同菜单项会对应不同姿势、表情与动作调整
- 背景存在“场景捕捉后再做扭曲和着色”的技术路线

### 0.2 推断性质的内容

文档里以下内容属于基于公开视频、截图和交互观感整理出的工程化推断：

- 入场时间轴中的具体秒数
- 每层元素的建议延迟、速度曲线与振幅
- 网页 / AE 中的替代实现方案

因此这份文档更适合用于“复刻和设计”，而不是作为官方逐帧设定稿。

## 1. 总体动效规则

先给一句总纲：

> P3R 的菜单不是“把 UI 放在角色旁边”，而是“让角色和界面一起沉入同一片心理海域”。

可以归纳成 6 条规则：

### 1.1 介质先于元素

先感受到水，再看见按钮、数字和人物。

所以入场时优先出现的是：

- 蓝色调压暗
- 波纹 / 扭曲
- 玻璃高光
- 漂浮颗粒

而不是按钮先跳出来。

### 1.2 大运动慢，小反馈快

- 角色、背景、流体、粒子：慢
- 选中态、按钮切换、数字响应：快

这保证了氛围是“失重”的，但操作仍然利落。

### 1.3 永远不要完全静止

P3R 的菜单成立，很大一部分原因是“定住后还在轻微呼吸”。

常驻微动通常来自：

- 发丝和衣摆的低频摆动
- 背景扭曲的连续波动
- 高光在边缘轻微游走
- 粒子向上或斜向漂移

### 1.4 软流体包裹硬信息

底层是液体、雾、光、玻璃。
上层是数字、标签、菜单、状态条。

两者并置，才会同时拥有：

- 柔软的心理空间
- 清晰的系统性信息

### 1.5 角色本身就是导航系统的一部分

角色不是插画背景，而是导视元素。

- 不同菜单会换姿态
- 视线和肢体会帮助引导注意力
- 角色和数字、标题、按钮之间会形成构图牵引

### 1.6 转场靠“浸入”而不是“切页”

P3R 更像从一个液体层滑入另一个液体层。

所以它的转场观感通常不是：

- 一个页面消失，另一个页面出现

而是：

- 原画面被染蓝
- 产生折射和拖拽
- 新信息像从同一片水体里浮现出来

## 2. 角色入场动作时间线

这一节专门回答你提到的“主角头后仰朝下，像浸入水面”的效果。

### 2.1 动作核心

这个入场不是攻击型登场，而是“失重下沉”。

它的动作关键词应该是：

- 放松
- 下沉
- 漂浮
- 被液体拖拽
- 美而脆弱

因此，不能做成：

- 猛冲进来
- 急停
- 大幅 bounce
- 过强的弹性 overshoot

### 2.2 推荐时间轴

下面的时间轴是适合网页或视频复刻的工程化版本，不是官方逐帧时码。

#### Phase A: 介质建立 `0.00s - 0.18s`

目标：先让观众感到“进入了另一层空间”。

画面变化：

- 当前画面或背景先被快速降饱和
- 蓝青色覆盖层压上来
- 轻微波纹开始生效
- 画面边缘先出现玻璃反光或液面亮带

建议参数：

- 色调变化：`120ms - 180ms`
- 波纹强度：小到中，不要一开始就剧烈扭曲
- 透明度曲线：快速进入，末端减速

#### Phase B: 主角破面入场 `0.10s - 0.42s`

目标：让角色像是“从液体另一侧浮现 / 沉入”。

推荐动作结构：

- 主角从画面上方或偏右上区域进入
- 头部略后仰，面部朝下或斜下
- 肩颈先进入视觉中心，躯干随后跟上
- 发丝和衣摆延迟跟随，表现水阻

推荐姿态特征：

- 头颈角度偏后打开，不是低头缩颈
- 身体主轴有轻微斜切，避免正面站直
- 手臂不要发力，保持漂浮感

推荐运动曲线：

- 位置：`easeOutCubic` 或 `cubic-bezier(0.16, 0.84, 0.24, 1)`
- 旋转：比位移更慢半拍
- 发丝 / 衣摆：再滞后半拍

避免的问题：

- 位移结束太硬，看起来像 UI 元素被平移进来
- 头部角度过大，会从“失重”变成“昏厥”
- 角色一进场就完全清晰，会失去“浸入感”

#### Phase C: 液体阻尼定格 `0.32s - 0.72s`

目标：角色到位，但像被水托住，不是硬停。

画面变化：

- 主角主体接近最终位置
- 速度明显下降
- 有非常轻的二次漂移
- 背景扭曲继续流动

这里建议做一个很小的“拖拽后回归”：

- 位移回摆：`6px - 18px`
- 旋转回摆：`1.5deg - 3deg`
- 时间：`180ms - 260ms`

重点是“阻尼”，不是“弹跳”。

#### Phase D: 信息层浮现 `0.42s - 0.90s`

目标：按钮、数字、标题像从同一介质里长出来。

推荐顺序：

1. 大标题 / 数字先出现
2. 主菜单按钮组滑入
3. 辅助状态信息补上
4. 底部操作提示最后锁定

每层的出现都应该带一点：

- 位移
- 遮罩揭示
- 亮度变化

而不只是纯透明度淡入。

#### Phase E: 常驻微动 `0.90s+`

菜单打开后，画面进入低频循环。

建议保留的持续动作：

- 主角整体上下浮动：`4px - 10px`
- 发丝摆动周期：`3s - 6s`
- 背景波纹周期：`2s - 4s`
- 高光滑移周期：`4s - 8s`
- 粒子漂浮速度：极慢

### 2.3 入场时最关键的五个感觉

如果你在复刻时只能保五件事，优先保这些：

1. 角色不是飞进来，而是被液体拖进来
2. 头、颈、肩的姿态要有“后开”的脆弱感
3. 发丝和衣摆必须滞后
4. 到位时要有阻尼，而不是直接停
5. UI 元素要像从同一水层中浮现

## 3. 界面分层顺序

P3R 的好看，不只是素材本身，而是层次非常清楚。

下面这套分层适合拿去做网页或动效工程实现。

### 3.1 推荐层级结构

从后到前，建议拆成 8 层。

### Layer 1: 基底场景层

内容：

- 原始场景截图
- 抽象背景图
- 渐变空间底图

职责：

- 提供空间纵深
- 让后续扭曲有参照物

动效：

- 几乎不自己动
- 主要交给上方的扭曲层处理

### Layer 2: 冷色滤镜层

内容：

- 蓝青渐变
- 去饱和
- 对比压缩
- 轻微 posterize 感

职责：

- 统一气质
- 把现实场景拉进“心理海域”

动效：

- 入场时快速建立
- 常驻阶段基本稳定

### Layer 3: 波纹 / 折射层

内容：

- UV 扭曲
- 正弦波偏移
- 热浪式折射
- 局部液面扰动

职责：

- 提供“水感”
- 让整个 UI 不像平面拼贴

动效：

- 持续低频循环
- 强度不要盖过可读性

### Layer 4: 角色主渲染层

内容：

- 主角身体
- UI 专用姿态
- 表情和视线

职责：

- 作为主构图中心
- 把视觉重心从“面板”转移到“人物+面板关系”

动效：

- 入场主动画
- 常驻漂浮和细节摆动

### Layer 5: 角色材质混合层

内容：

- 头发 / 衣物局部折射
- 高光边缘
- 与背景扭曲共享的材质语言

职责：

- 让角色“浸在 UI 里”
- 避免角色像单独贴图飘在最上面

动效：

- 和背景波纹不同步地轻微变化
- 强度低于背景主扭曲

### Layer 6: 主信息层

内容：

- 菜单按钮
- 章节标题
- 数字编号
- 页面标签

职责：

- 保证导航和阅读效率

动效：

- 快于角色
- 位移距离短
- 多用切片揭示、横向滑入、斜向掠过

### Layer 7: 辅助信息层

内容：

- 状态条
- 队伍信息
- 货币
- 操作提示

职责：

- 提供交互上下文
- 不抢主视觉

动效：

- 比主信息层更轻
- 延迟略后
- 稳定性高于装饰性

### Layer 8: 前景装饰层

内容：

- 高光划痕
- 泡粒
- 微小漂浮碎片
- 局部镜面闪烁

职责：

- 提升空气感和层次感

动效：

- 振幅小
- 透明度低
- 只做“存在感”，不做“主表演”

### 3.2 层与层之间的关系

最重要的不是层多，而是每层职责不能混。

### 正确关系

- 基底负责空间
- 滤镜负责气质
- 扭曲负责介质
- 角色负责情绪
- 信息层负责可用性
- 装饰层负责氛围

### 常见错误

- 把所有蓝色都压到同一层，结果画面发脏
- 扭曲层过强，导致字不可读
- 角色在最上层完全不受介质影响，看起来像海报
- 装饰粒子太多，画面变成“动态壁纸”
- 所有元素用同一种 easing，失去层次

### 3.3 速度分配建议

可以直接按这个原则配速：

- 背景扭曲：最慢
- 角色漂浮：慢
- 主标题 / 数字：中快
- 菜单按钮：快
- hover / focus 反馈：最快

也就是说，P3R 不是所有东西一起慢悠悠地动，而是：

- 氛围层慢
- 操作层快

## 4. 网页 / CSS / AE 复刻方法

这一节不追求 1:1 技术完全一致，而是给出三种可落地方案。

### 4.1 方案 A：纯网页轻量复刻

适用：

- Astro / React 页面
- 宣传页
- 作品集
- 不依赖 WebGL 的中轻量实现

### 结构建议

推荐 DOM 结构：

```html
<main class="p3r-menu">
  <div class="p3r-bg"></div>
  <div class="p3r-distortion"></div>
  <div class="p3r-character"></div>
  <div class="p3r-ui">
    <div class="p3r-title"></div>
    <nav class="p3r-nav"></nav>
    <div class="p3r-meta"></div>
  </div>
  <div class="p3r-fx"></div>
</main>
```

### 实现重点

#### 背景

- 用场景截图或抽象渐变做基底
- 叠加蓝青 radial-gradient 和 linear-gradient
- 用 `filter: saturate() contrast() brightness()` 做整体校色

#### 波纹

轻量网页里不要硬做真实流体模拟，优先用：

- 两层半透明伪元素
- `transform` + `background-position`
- SVG turbulence / displacement
- 极轻的 blur 和 mask

#### 角色

优先级从低到高：

1. PNG 序列 / 单张立绘 + 位移
2. 带 alpha 的视频
3. WebGL / Three.js 角色层

如果只是做观感复刻，单张立绘也够，但要补这些动作：

- `translateY`
- `translateX`
- `rotate`
- `scale`
- 发丝局部 mask 流动

#### UI 层

- 标题先出
- 菜单按钮做 stagger
- hover 反馈要短促
- 不要大面积 fade，尽量用滑入和切片揭示

### 推荐动画语言

网页中可以直接用这些规则：

```css
character-enter: 520ms ~ 760ms
ui-reveal: 240ms ~ 420ms
hover-response: 120ms ~ 180ms
idle-float: 3s ~ 6s infinite alternate
wave-loop: 2s ~ 4s linear infinite
```

### 网页实现时最容易翻车的点

- blur 太重，画面像毛玻璃后台
- 蓝色太纯，失去灰度层次
- 角色漂浮幅度太大，像屏保
- 按钮也跟着大幅缓动，导致操作发黏

### 4.2 方案 B：网页高保真复刻

适用：

- 作品展示页
- 交互 demo
- 希望更接近游戏质感

建议技术：

- WebGL / Three.js
- offscreen scene capture 或预渲染贴图
- displacement map
- mask-driven material blend

推荐做法：

1. 先准备一个被蓝化的背景纹理
2. 再用一张流动的 displacement map 驱动扭曲
3. 角色单独成层
4. 角色头发 / 衣物局部再吃一次较弱的扭曲
5. UI 文本层保持清晰，只在入场时受轻微遮罩影响

这类方案的关键不是“全都扭曲”，而是：

- 背景扭曲最明显
- 角色局部扭曲
- 文本只吃很少的动效

### 4.3 方案 C：After Effects 复刻

适用：

- 动效分镜
- UI pitch video
- 包装片头

### AE 图层建议

从下到上：

1. Background Plate
2. Blue Grade
3. Distortion Map
4. Character Base
5. Character Highlight
6. Title / Number
7. Navigation
8. Particle / Glass FX

### AE 效果建议

- `Displacement Map`：做整体水波扭曲
- `Turbulent Displace`：做局部液面扰动
- `Glow`：只用于高光边缘，不要全局打满
- `Directional Blur`：给切片掠过或数字拖影
- `Posterize Time`：谨慎使用，只适合个别图形化层

### 角色入场在 AE 里的拆法

角色至少拆成 3 组：

1. 头部与上躯干
2. 发丝前层
3. 发丝后层 / 衣摆

这样可以做出：

- 本体先到位
- 发丝延迟跟随
- 局部高光后到

推荐关键帧逻辑：

- 主体位移：先快后慢
- 主体旋转：比位移晚到位
- 发丝摆动：再晚一点回落
- 最后进入低频漂浮循环

### AE 里最重要的控制

不要把所有层都挂同一个 `wiggle`。

正确做法是区分：

- 主体漂浮
- 发丝拖尾
- 高光滑移
- 背景扭曲

每个系统的频率和振幅都不同，才像 P3R。

## 5. 一份可直接执行的复刻规范

如果你要把这套东西交给前端或动效师，可以直接给下面这份简版。

### 5.1 关键词

- 冷
- 透明
- 失重
- 玻璃感
- 柔性介质
- 快速反馈

### 5.2 入场规则

- 先建立液体介质，再出现角色和 UI
- 角色从上方或斜上方浸入，不做硬切平移
- 头颈肩用后开姿态，避免直立站姿
- 发丝和衣摆必须滞后
- 到位后保留轻微阻尼和持续微动

### 5.3 界面规则

- 背景慢，角色慢，按钮快
- 标题先于按钮
- 主信息强于辅助信息
- 文本必须保持清晰，不为氛围牺牲可读性
- 装饰只增强空间，不抢主视觉

### 5.4 技术规则

- 轻量网页：渐变 + 遮罩 + 位移 + 轻扭曲
- 高保真网页：displacement + 分层材质 + 局部 mask
- AE：位移、旋转、发丝延迟、扭曲图、玻璃高光分层处理

## 6. 最后一句判断标准

如果你做出来的效果更像：

- “角色很帅地进场，UI 跟着淡入”

那还不够 P3R。

如果更像：

- “角色、数字、按钮和背景一起沉进一层冷蓝的心理海面里”

那方向就对了。
